New to Safari extensions?

  1. Open Safari.
  2. Choose Safari › Settings… (⌘,), then click Extensions.
  3. Check the box next to Map Path.
  4. When prompted about website access, choose Always Allow on Every Website.

Open Settings jumps to Map Path's settings (turn on Allow Extension + All Websites: Allow). Test it now opens a Safari test page — if a tap on a Google Maps link opens Apple Maps, your setup is complete. If it still opens Google or Waze, the most likely fix is All Websites: Allow (not Ask).

Important — please read. Apple lets you, not us, set up extension permissions. That's a privacy and security feature. If you don't complete all of these steps, Safari blocks Map Path and your links will still open in Google, Waze, or the other map sites.

3 steps in iOS Settings

  1. Turn on Allow Extension. In Settings → Apps → Safari → Extensions, tap Map Path and switch Allow Extension on.
  2. Turn on Allow Extension. In Settings → Safari → Extensions, tap Map Path and switch Allow Extension on.
  3. Set All Websites to Allow. Same screen, under Permissions, tap All Websites and choose Allow. If this is left on Ask, Map Path can't run and clicks still open the original map service.
  4. Optional — Private Browsing. If you want map links to route to Apple Maps inside Safari's Private tabs too, also turn on Allow in Private Browsing. Heads up about privacy: Apple Maps keeps its own Recents list, independent of Safari. A map link you tap from a Private Safari tab will still show up in Apple Maps' Recents. Map Path can't change that — only Apple controls Maps' history. If that matters to you, clear Maps' Recents from inside the Maps app.
